## 수강생용 슬라이드의 구성 원칙

각 회차는 실무자가 **AI 툴을 실제로 어떻게 다루고 어떤 방향으로 쓸지**에 초점을 맞췄습니다.

- **학습 목표 → 오늘의 흐름(120분) → 개념 → 「오늘의 툴 다루기」 → 실습 → 핵심 3줄 → 미션 안내** 순서로 흐릅니다.
- 「오늘의 툴 다루기」 카드는 도구별로 *이렇게 쓴다 / 무엇을 보나 / 주의점*을 명시해, 툴을 "답 얻는 기계"가 아니라 "브랜드를 측정·개선하는 계측기"로 다루도록 설계했습니다.
- 모든 실습·프롬프트는 그대로 복사해 쓸 수 있는 형태입니다.

## 강사용 가이드의 구성 (수강생 배포 금지)

강사가 수업을 그대로 운영할 수 있도록, 회차마다 다음을 담았습니다.

1. **운영 타임라인 × 시연 매핑** — 분 단위 진행표에 시연 번호(D1~D5) 연결
2. **시연 시나리오 D1~D5** — 강사가 보여줄 예시 화면을 유형별로 분류
   - `웹검색` · `사이트` · `유튜브` · `뉴스` · `라이브 시연` 태그로 구분
   - 각 시연마다 **링크/검색어**와 **"이 말을 하세요" 시연 포인트**를 명시
3. **핵심 시연 상세** — 하이라이트 시연의 진행 순서
4. **시연 백업 플랜** — 라이브 실패·결과 변동 시 대응 (실패를 교육 소재로 전환)
5. **미션 배포문** — 공유 시트에 그대로 게시할 수 있는 과제 안내문
6. **미션 평가 루브릭** — 항목·배점·우수/미흡 기준
7. **토론 유도 질문 & 예상 질문 대응** — 강사 답변 가이드 포함
8. **수업 전 준비 체크리스트**

### 시연 자료 유형 태그

| 태그 | 의미 | 예시 |
|---|---|---|
| `웹검색` | 검색엔진에서 라이브 검색 | 구글 AI Overviews, site: 검색, PAA |
| `사이트` | 특정 웹사이트/도구 화면 | Perplexity 출처, 공식 문서, 서치어드바이저 |
| `유튜브` | 유튜브 영상/채널 | Google Search Central 채널 |
| `뉴스` | 뉴스 검색·기사 | 제로클릭 검색, AI 검색 트래픽 기사 |
| `라이브 시연` | AI 툴 실시간 실행 | 프롬프트 실행, 라이브 리라이트 |

> 강사용 슬라이드의 모든 외부 링크·검색어는 **수업 전날 접속·노출 여부를 반드시 확인**하세요. AI 검색 결과는 날짜·계정·지역·도구에 따라 달라집니다. 백업 스크린샷은 필수입니다.

## 디자인 시스템 — McKinsey 스타일

컨설팅 프레젠테이션 관례를 적용해 신뢰감과 가독성을 높였습니다. (참고: `seulee26/mckinsey-pptx`의 디자인 토큰)

- **팔레트**: 딥 네이비 `#0A1F3D`·`#0F2A4A` + 블루 스케일(`#2E9BD6`·`#1F6FA8`·`#4FB2E5`), 신호등 상태색(초록 `#4CAF50` / 앰버 `#E0A44A` / 빨강 `#E04E5E`).
- **폰트**: Arial(영문·숫자) + Noto Sans KR(한글) 페어링.
- **슬라이드 관례**:
  - **커버**: 딥 네이비 풀블리드 + 블루 막대 모티프.
  - **콘텐츠**: 상단 kicker(회차·주제) + 제목 + 얇은 언더룰, 우상단 **트래커**(강사용은 `강사용` 배지), 하단 **소스라인 + 카피라이트 + 페이지**.
  - **강조 슬라이드**: 네이비 풀스크린 어서션(핵심 메시지).
  - **표**: 네이비 헤더 + 회색 줄무늬(zebra), 본문 12pt.
  - **시연 카드**: 밀집 표 대신 풀폭 카드 2슬라이드(유형 색태그 + 검색어 모노 + 멘트).
- 카테고리 색상은 실제 분류(웹검색/사이트/유튜브/뉴스/라이브 시연 등)를 인코딩하되 McKinsey 팔레트 톤으로 정렬했습니다.
- 수강생용/강사용 모두 동일한 McKinsey 시스템을 쓰고, 강사용은 `강사용` 배지로 구분합니다.

## 편집 방법

- 파일은 표준 `.pptx`로, PowerPoint · Keynote · Google Slides · LibreOffice Impress에서 모두 열립니다.
- 폰트: Arial + Noto Sans KR (없는 환경에서는 시스템 대체 폰트로 렌더링)
- 팔레트/폰트를 바꾸려면 `_build/mck_helpers.py` 상단의 색상 상수·`LAT`/`EA` 폰트명을 수정 후 재생성하세요.

## 재생성 (선택)

슬라이드는 `python-pptx`로 코드 생성했습니다. 콘텐츠(`build_student.py`·`build_instructor.py`)와
McKinsey 시각 레이어(`mck_helpers.py`)가 분리돼 있어, 내용만 고치고 스타일은 그대로 재빌드할 수 있습니다.

```bash
pip install python-pptx
cd _build
# McKinsey 스타일로 12개 덱 전체 생성
OUT_S="../수강생용" OUT_I="../강사용" python3 build_mck.py
```

- `mck_helpers.py` — McKinsey 디자인 시스템(팔레트·폰트·builder). 색/폰트 변경은 여기.
- `build_student.py` · `build_instructor.py` — 회차별 콘텐츠 데이터(빌더 무관).
- `helpers.py` — 이전 인디고/클레이 시스템(참고용 보관).
