# AI 시대 마케팅, 누가 살아남을까?

6주 실무 스터디 강의안  
대상: 실무 마케터 초중급  
운영: 주 1회, 회당 2시간, 총 6회  
작성 기준일: 2026-07-03

---

## 1. 과정 포지셔닝

이 강의의 핵심 질문은 "AI가 콘텐츠를 대신 만드는 시대에 마케터의 가치는 어디서 생기는가?"이다. 답은 단순한 툴 사용법이 아니라, 검색과 발견의 구조 변화에 맞춰 브랜드가 인용되고, 추천되고, 기억되도록 만드는 능력이다.

기존 마케팅은 대체로 노출, 클릭, 전환의 흐름으로 설계됐다. AI 검색 이후에는 사용자가 링크를 클릭하기 전에 Google AI Overviews, AI Mode, ChatGPT Search, Perplexity, 네이버 검색/콘텐츠 생태계 안에서 답을 먼저 받는다. 따라서 실무 마케터는 SEO를 버리는 것이 아니라, SEO를 기반으로 AEO와 GEO까지 확장해야 한다.

이 과정에서는 GEO를 지리 기반 최적화가 아니라 Generative Engine Optimization, 즉 생성형 검색/답변 엔진에서 브랜드와 콘텐츠가 인용·참조·추천되도록 만드는 실무로 정의한다.

### 6주 후 수강생 산출물

- 내 브랜드/서비스의 AI 검색 가시성 진단표
- SEO/AEO/GEO 통합 콘텐츠 1개
- 네이버/구글/ChatGPT/Perplexity 기준 프롬프트 리서치 로그
- 콘텐츠 리라이트 전후 비교표
- AI 활용 마케팅 업무 자동화 미니 워크플로우
- 2~3년 커리어/조직 준비 로드맵

## 3. 회차별 상세 강의안

### 1회차. AI가 바꾼 발견의 구조

목표: 검색, 추천, AI 답변, 커뮤니티, 숏폼이 하나의 발견 경로로 합쳐지는 변화를 이해한다.

120분 운영:

| 시간 | 내용 |
|---|---|
| 0~10분 | 오프닝: "검색 순위 1위인데 매출이 안 늘어나는 이유" |
| 10~30분 | AI 검색의 변화: Google AI Overviews, AI Mode, ChatGPT Search, Perplexity |
| 30~50분 | SEO/AEO/GEO 개념 구분: 순위, 답변, 인용 |
| 50~70분 | 국내 시장 맥락: 네이버 검색, 블로그/카페/쇼핑, 네이버 서치어드바이저 |
| 70~95분 | 실습: 내 브랜드를 AI에게 물어보기 |
| 95~110분 | 그룹 리뷰: "AI는 우리를 어떻게 오해하는가?" |
| 110~120분 | 과제 안내 |

핵심 메시지:

- AI 검색은 사용자의 질문을 더 길고 구체적으로 만든다.
- 클릭 전 단계에서 답이 제공되므로, 순위보다 "답변 내 존재감"이 중요해진다.
- 마케터는 키워드 관리자에서 질문·근거·맥락 설계자로 이동해야 한다.

실습: AI 검색 가시성 진단

1. ChatGPT, Perplexity, Gemini, Google, 네이버에서 동일 질문을 입력한다.
2. 브랜드명 직접 질문, 카테고리 추천 질문, 비교 질문, 문제 해결 질문을 각각 테스트한다.
3. 결과를 아래 기준으로 기록한다.

| 프롬프트 | 내 브랜드 언급 | 경쟁사 언급 | 인용 출처 | 오류/누락 | 다음 액션 |
|---|---:|---:|---|---|---|
| "OO 추천해줘" |  |  |  |  |  |
| "OO와 경쟁사 비교" |  |  |  |  |  |
| "초보자가 OO 고르는 법" |  |  |  |  |  |

과제: 자사/클라이언트 브랜드의 AI 검색 질문 20개를 만들고 결과를 기록한다.

### 2회차. SEO의 재정의: AI 시대에도 기본기가 이긴다

목표: 검색엔진이 콘텐츠를 발견, 크롤링, 색인, 이해하도록 만드는 기본기를 실무 체크리스트로 만든다.

120분 운영:

| 시간 | 내용 |
|---|---|
| 0~15분 | 과제 리뷰: AI가 우리 브랜드를 말하는 방식 |
| 15~35분 | Google/Naver 검색엔진의 기본 구조 |
| 35~55분 | 기술 SEO: 색인, robots.txt, 사이트맵, 내부링크, JS 콘텐츠 |
| 55~75분 | 콘텐츠 SEO: 제목, 구조, 중복, 검색 의도, 신뢰 신호 |
| 75~105분 | 실습: 내 콘텐츠 SEO 응급 진단 |
| 105~120분 | 리라이트 방향 정리 |

핵심 메시지:

- AI 검색에 등장하려면 먼저 검색엔진이 콘텐츠를 접근·색인할 수 있어야 한다.
- Google은 AI 기능에 별도 "비밀 최적화"가 있는 것이 아니라, 기본 SEO와 도움이 되는 콘텐츠를 강조한다.
- 네이버는 Yeti 크롤러 접근, 웹마스터도구 등록, 사이트맵/RSS, 표준 HTML 링크, 중복 콘텐츠 관리가 중요하다.

실습: 기술/콘텐츠 SEO 체크리스트

| 항목 | 점검 질문 | 상태 |
|---|---|---|
| 색인 | Google/Naver에서 `site:도메인` 검색 결과가 있는가? |  |
| 크롤링 | robots.txt가 Googlebot/Yeti를 막고 있지 않은가? |  |
| 사이트맵 | XML 사이트맵과 RSS가 제출되어 있는가? |  |
| 내부링크 | 중요한 페이지가 HTML 링크로 연결되어 있는가? |  |
| 제목 | 각 페이지 제목이 중복되지 않고 검색 의도를 담는가? |  |
| 구조 | H1/H2/H3가 논리적이고 답변 추출이 쉬운가? |  |
| 신뢰 | 작성자, 날짜, 출처, 근거, 실제 경험이 드러나는가? |  |
| 전환 | 방문자가 다음 행동을 이해할 수 있는가? |  |

과제: 기존 콘텐츠 1개를 선택해 SEO 문제 5개와 수정안을 작성한다.

### 3회차. AEO: 답변으로 뽑히는 콘텐츠 만들기

목표: 질문형 검색과 AI 답변에 적합한 콘텐츠 구조를 익힌다.

120분 운영:

| 시간 | 내용 |
|---|---|
| 0~15분 | SEO 진단 과제 공유 |
| 15~35분 | AEO 개념: Featured Snippet, People Also Ask, AI 답변 |
| 35~55분 | 좋은 답변 블록의 구조: 직접 답변, 조건, 예외, 표, 단계 |
| 55~75분 | 질문 클러스터 만들기: 정의, 비교, 추천, 비용, 방법, 실패 |
| 75~105분 | 실습: 기존 글을 AEO형으로 재구성 |
| 105~120분 | 피드백: 답변이 짧지만 얕지 않은가? |

핵심 메시지:

- AEO는 FAQ를 많이 넣는 일이 아니라, 사용자의 실제 질문에 검증 가능한 답을 제공하는 구조화 작업이다.
- 좋은 답변은 "한 문장 결론 + 조건 + 근거 + 다음 행동"을 가진다.
- AI는 문서 전체보다 잘 쪼개진 의미 단위, 표, 비교, 단계형 설명을 활용하기 쉽다.

실습: AEO 콘텐츠 블록 템플릿

```text
질문:
한 문장 답:
이 답이 맞는 조건:
예외/주의:
근거:
실무 예시:
다음 행동:
관련 질문:
```

예시:

```text
질문: AI 시대에도 블로그 SEO를 해야 하나?
한 문장 답: 해야 한다. 다만 키워드 반복보다 검색 의도, 구조, 원문성, 인용 가능성을 강화해야 한다.
조건: 제품/서비스 탐색, 비교, 문제 해결형 검색 수요가 있는 경우.
예외: 단기 이벤트성 캠페인처럼 검색 누적 가치가 낮은 경우.
근거: Google은 AI 기능에서도 기본 SEO와 접근 가능한 콘텐츠를 강조한다.
실무 예시: "가격", "비교", "사용법", "실패 사례" 페이지를 질문형 구조로 재작성한다.
다음 행동: 기존 상위 10개 콘텐츠를 질문 단위로 쪼개고 누락 질문을 보강한다.
```

과제: 기존 콘텐츠 1개에 질문형 블록 10개를 추가한다.

### 4회차. GEO: 생성형 엔진에 인용되는 브랜드 만들기

목표: ChatGPT Search, Perplexity, Google AI 기능 등 생성형 엔진에서 인용될 가능성을 높이는 콘텐츠 조건을 설계한다.

120분 운영:

| 시간 | 내용 |
|---|---|
| 0~15분 | AEO 과제 리뷰 |
| 15~35분 | GEO 개념과 연구 흐름 |
| 35~55분 | 인용 가능 콘텐츠의 조건: 명확성, 구조, 출처, 원문성, 비교 가능성 |
| 55~75분 | LLM이 브랜드를 인식하는 경로: 웹문서, 언론, 리뷰, 커뮤니티, 데이터베이스 |
| 75~105분 | 실습: GEO 리라이트 |
| 105~120분 | AI 검색 재테스트 |

핵심 메시지:

- GEO는 "AI를 속이는 기술"이 아니라, AI가 안전하게 인용할 수 있는 공개 근거를 늘리는 작업이다.
- 연구들은 콘텐츠 구조, 출처 명시, 통계, 인용, 명확한 문서 설계가 생성형 엔진 가시성에 영향을 줄 수 있음을 보여준다.
- LLM 답변은 같은 질문도 실행 시점, 표현, 도구, 모델에 따라 달라지므로 반복 측정이 필요하다.

GEO 리라이트 체크리스트:

| 항목 | 리라이트 방식 |
|---|---|
| 원문성 | 우리만 말할 수 있는 데이터, 사례, 고객 질문, 실패 경험을 넣는다. |
| 명시성 | "누구에게 적합한가/부적합한가"를 분명히 쓴다. |
| 비교성 | 대안, 경쟁 옵션, 선택 기준을 표로 정리한다. |
| 출처성 | 통계, 법규, 인용, 조사 방법을 표시한다. |
| 구조성 | H2/H3, 표, 리스트, 요약문, FAQ를 사용한다. |
| 최신성 | 작성일, 업데이트일, 버전, 변경 이력을 남긴다. |
| 엔티티 | 브랜드명, 제품명, 인물, 지역, 카테고리를 일관되게 표기한다. |

실습 프롬프트:

```text
너는 AI 검색 최적화 컨설턴트다.
아래 콘텐츠를 ChatGPT Search, Perplexity, Google AI Overviews에 인용될 가능성이 높도록 진단해줘.
기준은 1) 명확한 답변성 2) 출처/근거성 3) 구조화 4) 독창적 경험 5) 비교 가능성 6) 최신성이다.
각 항목을 5점 만점으로 평가하고, 고칠 문장과 추가해야 할 섹션을 제안해줘.

[콘텐츠 붙여넣기]
```

과제: 리라이트 전후 콘텐츠를 AI 검색에 다시 물어보고 변화 기록.

### 5회차. 실무에 써먹는 AI 마케팅 워크플로우

목표: AI를 단순 카피 생성기가 아니라 리서치, 기획, 제작, 검수, 분석 파트너로 쓰는 반복 업무 체계를 만든다.

120분 운영:

| 시간 | 내용 |
|---|---|
| 0~15분 | GEO 재테스트 결과 공유 |
| 15~35분 | AI 활용 원칙: 초안은 AI, 판단은 사람, 근거는 데이터 |
| 35~55분 | 리서치 워크플로우: 시장, 고객, 경쟁사, 검색 의도 |
| 55~75분 | 콘텐츠 워크플로우: 브리프, 초안, 검수, 재가공 |
| 75~100분 | 캠페인 워크플로우: 소재 가설, A/B 테스트, 성과 해석 |
| 100~120분 | 개인별 자동화 설계 |

실무 워크플로우 1: 콘텐츠 브리프 생성

```text
역할: 너는 B2B/B2C 콘텐츠 전략가다.
목표: [제품/서비스]의 검색 유입과 AI 답변 인용 가능성을 동시에 높이는 콘텐츠 브리프를 만든다.
대상 고객: [고객]
핵심 문제: [문제]
경쟁 콘텐츠: [URL 또는 요약]

다음 형식으로 작성해줘.
1. 핵심 검색 의도
2. 질문 클러스터 20개
3. 반드시 포함할 근거/데이터
4. H2/H3 구조
5. AEO 답변 블록 5개
6. GEO 인용 가능성 강화 요소
7. 제목 10개
8. CTA 3개
```

실무 워크플로우 2: 콘텐츠 검수

```text
아래 콘텐츠를 실무 편집장 관점에서 검수해줘.
목표는 SEO/AEO/GEO 동시 개선이다.

검수 기준:
- 검색 의도와 제목이 일치하는가
- 첫 300자 안에 명확한 답이 있는가
- 근거 없는 과장이 있는가
- AI가 인용하기 좋은 표/목록/정의가 있는가
- 네이버 블로그/웹문서/구글 검색 모두에서 어색하지 않은가
- 전환 CTA가 자연스러운가

출력:
1. 총평
2. 반드시 고칠 부분
3. 추가하면 좋은 섹션
4. 문장 단위 수정 예시
5. 발행 전 체크리스트
```

실무 워크플로우 3: 성과 해석

```text
아래 성과 데이터를 보고 다음 달 액션을 제안해줘.
데이터: [노출, 클릭, CTR, 평균순위, 체류시간, 전환, 검색어, 유입채널]
목표: [리드/구매/예약/문의]

분석 형식:
1. 가장 큰 문제 3개
2. SEO 문제인지, SERP/AI 답변 문제인지, 콘텐츠 문제인지 구분
3. 당장 수정할 콘텐츠 5개
4. 새로 만들 콘텐츠 5개
5. 측정해야 할 AI 검색 프롬프트 10개
```

과제: 자신의 반복 업무 하나를 AI 워크플로우로 바꾸고, 단계별 프롬프트를 만든다.

### 6회차. 살아남는 마케터의 2~3년 준비

목표: AI가 대체하기 어려운 마케터 역량과 포트폴리오 전략을 정리한다.

120분 운영:

| 시간 | 내용 |
|---|---|
| 0~20분 | 5주간 산출물 리뷰 |
| 20~45분 | 사라지는 업무와 남는 업무 |
| 45~65분 | 2026~2029 마케팅 변화 시나리오 |

AI 시대에도 남는 마케터 역량:

| 역량 | 설명 | 포트폴리오 증거 |
|---|---|---|
| 문제 정의 | AI에게 시킬 일을 정확히 정의한다. | 리서치 브리프, 고객 인사이트 문서 |
| 판단력 | AI 결과의 오류, 과장, 법적 리스크를 잡는다. | 검수 체크리스트, 수정 전후 |

| 기간 | 개인 마케터 | 팀/조직 |
|---|---|---|

| 6~12개월 | 검색·AI·커뮤니티 통합 KPI 운영 | CRM/GA/GSC/광고 데이터 연결, 실험 문화 구축 |

| 2~3년 | 전략가형 마케터로 전환: 문제 정의, 실험, 판단 | AI가 운영을 맡고 사람이 전략·브랜드·윤리를 책임지는 구조 |

## 4. 강의용 핵심 프레임워크

### SEO/AEO/GEO 한 장 정리

| 구분 | 목표 | 주요 화면 | 콘텐츠 조건 | 측정 지표 |
|---|---|---|---|---|
| SEO | 검색 결과에서 발견된다 | 구글/네이버 검색결과 | 크롤링, 색인, 검색 의도, 품질 | 노출, 클릭, 순위, CTR |
| AEO | 질문에 대한 답으로 뽑힌다 | 스니펫, Q&A, AI 답변 | 직접 답변, 구조화, FAQ, 표 | 답변 노출, 스니펫, 질문 커버리지 |
| GEO | 생성형 엔진에 인용·추천된다 | AI Overviews, ChatGPT, Perplexity | 출처성, 독창성, 엔티티, 비교, 최신성 | 브랜드 언급, 인용 URL, 경쟁사 대비 점유 |

### 콘텐츠를 AI 친화적으로 바꾸는 7단계

1. 타깃 고객의 질문 30개를 모은다.
2. 질문을 정의/비교/추천/비용/방법/문제/후기로 분류한다.
3. 각 질문에 한 문장 답을 먼저 쓴다.
4. 표, 단계, 체크리스트로 구조화한다.
5. 출처, 기준일, 작성자, 실제 사례를 넣는다.
6. 브랜드명/제품명/카테고리명을 일관되게 표기한다.
7. AI 검색 도구에서 반복 테스트하고 로그를 남긴다.

### AI 검색 가시성 KPI

| KPI | 정의 | 측정 주기 |
|---|---|---|
| Brand Mention Rate | 핵심 프롬프트 중 브랜드가 언급된 비율 | 주간/월간 |
| Citation Rate | 브랜드 사이트가 출처로 인용된 비율 | 월간 |
| Competitor Share | 경쟁사 대비 언급 점유율 | 월간 |
| Answer Accuracy | AI 답변의 사실 오류 수 | 월간 |
| Prompt Coverage | 구매 여정별 질문 커버리지 | 월간 |
| Assisted Conversion | AI/검색 콘텐츠 접점 이후 전환 | 월간 |
| Content Refresh Velocity | 업데이트된 핵심 콘텐츠 수 | 월간 |

| Jasper / Copy.ai / HubSpot Breeze 등 마케팅 특화 AI | 마케팅 템플릿, 캠페인/CRM/세일즈 워크플로우 | 이메일, 랜딩페이지, 세일즈 카피, CRM 기반 자동화 | 범용 AI보다 유연성은 낮을 수 있으나 운영 편의성은 높음 |
| Notion AI / Canva AI / Gamma / Beautiful.ai | 문서, 디자인, 발표자료 제작 | 강의자료, 제안서, SNS 카드뉴스, 내부 문서 | 최종 톤앤매너와 브랜드 가이드 수동 검수 |

선택 기준:

### 6-2. 마케터가 알아야 할 MCP

MCP(Model Context Protocol)는 AI 애플리케이션이 외부 데이터, 도구, 업무 시스템에 표준 방식으로 연결되도록 하는 규격이다. 쉽게 말해 AI를 "말만 하는 챗봇"에서 "파일을 읽고, DB를 조회하고, 툴을 실행하고, 워크플로우를 수행하는 업무 파트너"로 바꾸는 연결 규칙이다.

마케터 관점에서 MCP가 중요한 이유:

- AI가 사내 자료, 브랜드 가이드, 상품 DB, CRM, 광고 성과 데이터, 검색 콘솔 데이터에 접근할 수 있다.
- 한 번 만든 MCP 서버를 Claude, ChatGPT, 개발도구, 로컬 AI 클라이언트 등 여러 환경에서 재사용할 수 있다.
- "프롬프트 잘 쓰기"에서 "AI가 쓸 수 있는 데이터와 도구를 설계하기"로 역량이 이동한다.

마케팅 MCP 예시:

| MCP 서버 | 연결 데이터/도구 | 가능한 업무 |
|---|---|---|
| Brand Guide MCP | 브랜드 톤, 금칙어, 메시지 하우스 | 카피 검수, 브랜드 일관성 체크 |
| Content Library MCP | 블로그, 랜딩페이지, FAQ, 백서 | 기존 콘텐츠 재활용, 중복 방지, 내부링크 추천 |
| SEO MCP | GSC, GA4, 네이버 서치어드바이저, 키워드 DB | 트래픽 하락 원인 분석, 리라이트 우선순위 |
| Ads MCP | Google Ads, Meta Ads, 네이버 검색광고 | 소재 성과 분석, 예산 이동 제안 |
| CRM MCP | HubSpot, Salesforce, Notion, Airtable | 리드 세그먼트별 메시지 생성, 영업 인사이트 |
| Commerce MCP | Shopify, 카페24, 스마트스토어, 쿠팡 | 상품 설명 개선, 리뷰 분석, 재고/판매 기반 캠페인 |

수업 실습 아이디어:

```text
우리 팀의 마케팅 업무에서 AI가 접근하면 좋은 데이터 10개를 적어라.
각 데이터에 대해 1) 민감도 2) 담당자 3) 업데이트 주기 4) AI가 수행할 업무 5) 연결 허용 여부를 표시하라.
```

MCP 도입 판단:

| 상황 | 권장 |
|---|---|
| 개인 실습, 공개 자료 기반 | MCP 없이 ChatGPT/Gemini/Claude/Perplexity로 충분 |
| 반복적으로 같은 사내 자료를 붙여넣음 | 커넥터 또는 MCP 검토 |
| 여러 AI 도구에서 같은 데이터/도구를 쓰고 싶음 | MCP 적합 |
| 고객 개인정보/매출/계약 데이터 포함 | 권한, 로그, 비식별화, 승인 체계 먼저 |
| 개발 리소스 없음 | Notion/Google Drive/Slack 등 기본 커넥터부터 시작 |

### 6-3. 오프라인/사내 AI 구축

오프라인 AI 구축은 "인터넷이 안 되는 곳에서 ChatGPT 같은 것을 만든다"라기보다, 민감한 데이터를 외부 모델로 보내지 않고 로컬 또는 사내 인프라에서 모델과 검색 시스템을 운영하는 선택지다.

구축 단계:

| 단계 | 구성 | 적합한 상황 |
|---|---|---|
| 1단계: 로컬 앱 | LM Studio, Ollama, Jan, AnythingLLM | 개인 실험, 사내 보안 데모, 작은 문서 질의응답 |
| 2단계: 로컬 RAG | 로컬 모델 + 벡터DB + 문서 인덱싱 | 브랜드 가이드, FAQ, 제안서, 고객응대 스크립트 검색 |
| 3단계: 사내 API 서버 | llama.cpp 서버, vLLM, TGI, SGLang | 여러 팀이 같은 모델을 API로 사용 |
| 4단계: 사내 AI 업무 포털 | 인증, 권한, 로그, 프롬프트 템플릿, 검수 워크플로우 | 개인정보/계약/매출 데이터가 많은 조직 |
| 5단계: 파인튜닝/지속학습 | LoRA, SFT, 평가 데이터셋, 배포 파이프라인 | 특정 문체, 분류, 상담, 추출 작업을 반복 수행 |

기본 구성도:

```text
사용자
  -> 사내 AI 포털 또는 채팅 UI
  -> 권한 확인
  -> 검색/RAG: 문서, FAQ, CRM, 성과 데이터
  -> 로컬/사내 LLM: Llama, Qwen, Mistral, DeepSeek 계열 등
  -> 검수/로그/피드백
  -> 답변 또는 업무 실행
```

로컬 구축 도구:

| 도구 | 역할 | 마케터 관점 |
|---|---|---|
| Ollama | 로컬 오픈 모델 실행 | 가장 빠른 실험용. 모델 다운로드 후 명령어로 실행 |
| LM Studio | GUI 기반 로컬 모델 실행, 문서 질의응답, OpenAI 호환 로컬 서버 | 비개발자도 테스트하기 좋음 |
| llama.cpp | C/C++ 기반 경량 추론, GGUF 모델, 다양한 하드웨어 | 개발팀이 로컬/엣지 배포할 때 유용 |
| vLLM | 고성능 LLM 서빙, OpenAI 호환 API 서버, 대량 요청 처리 | 사내 서비스나 여러 사용자가 쓰는 AI 서버에 적합 |
| Open WebUI / AnythingLLM | 로컬/사내 챗봇 UI와 RAG | 사내 지식 챗봇 프로토타입 |
| Chroma / Qdrant / Weaviate / Milvus | 벡터DB | 문서 검색과 RAG의 저장소 |

오프라인 AI 구축 전 체크리스트:

| 체크 | 질문 |
|---|---|
| 목적 | 단순 보안 때문인가, 비용 절감인가, 지연시간인가, 커스터마이징인가? |
| 데이터 | 어떤 문서를 넣을 것인가? 최신성은 누가 관리하는가? |
| 보안 | 개인정보/영업비밀/계약정보가 포함되는가? 접근 권한은 어떻게 나눌 것인가? |
| 품질 | 답변 정확도는 누가 평가하고, 기준 데이터셋은 있는가? |
| 비용 | GPU, 서버, 운영 인력, 백업, 모니터링 비용을 계산했는가? |
| 법무 | 모델 라이선스, 저작권, 고객 데이터 처리 계약을 확인했는가? |
| 운영 | 모델 업데이트, 장애 대응, 로그 보관, 사용자 교육 계획이 있는가? |

실무 결론:

- 개인 마케터는 ChatGPT/Gemini/Claude/Perplexity를 능숙하게 쓰는 것이 먼저다.
- 팀 리더는 데이터 연결, 보안 기준, 검수 프로세스를 만들어야 한다.
- 조직은 "무조건 자체 구축"보다 공개 AI, 기업용 AI, 로컬 AI, 사내 RAG를 업무별로 섞는 하이브리드 전략이 현실적이다.
- 2~3년 후 경쟁력은 특정 AI 이름을 아는 것이 아니라, 데이터와 도구를 안전하게 연결하고 성과를 측정하는 운영 설계 능력에서 나온다.

## 7. 강사용 운영 팁

- 매회 첫 15분은 과제 리뷰로 시작한다. 이 과정의 성과는 강의 내용보다 수강생이 자기 브랜드를 계속 테스트하는 데서 나온다.
- AI 툴 결과를 정답처럼 보여주지 말고, "왜 이런 답이 나왔을까?"를 토론하게 한다.
- 실습 브랜드는 수강생 본인 회사가 가장 좋다. 민감하면 가상의 브랜드나 공개 브랜드로 대체한다.
- 수업 중 생성된 프롬프트와 결과는 모두 시트에 남긴다. 6주 후 이것이 포트폴리오가 된다.
- 검색/AI 결과는 날짜, 계정, 지역, 도구에 따라 달라질 수 있으므로 기준일을 항상 기록한다.
- 의료, 금융, 법률 등 고위험 산업은 AI 답변 사용보다 검수, 출처, 고지, 내부 승인 프로세스를 더 강조한다.

## 8. 첫 강의 오프닝 스크립트

"오늘의 질문은 단순합니다. AI 시대에 마케터는 사라질까요? 제 답은 반은 맞고 반은 틀리다입니다. 카피를 대량으로 찍어내는 사람, 검색어만 반복하는 사람, 리포트를 숫자 나열로 끝내는 사람은 위험합니다. 반대로 고객의 질문을 정의하고, AI가 인용할 만한 근거를 만들고, 브랜드의 관점을 구조화하고, 실험으로 성과를 증명하는 마케터는 더 중요해집니다.

6주 동안 우리는 AI 툴 사용법만 배우지 않습니다. 내 브랜드가 AI에게 어떻게 보이는지 측정하고, 검색엔진이 이해할 수 있는 구조로 고치고, 답변 엔진이 가져갈 수 있는 문장을 만들고, 생성형 엔진이 인용할 수 있는 근거 자산을 만들 겁니다. 이 과정을 끝내면 여러분 손에는 멋진 이론이 아니라, 바로 발행하고 측정할 수 있는 실무 자료가 남아야 합니다."
