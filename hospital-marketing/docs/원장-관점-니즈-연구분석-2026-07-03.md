# 원장 관점 니즈 연구분석 — 2026-07-03

방법: 제품 전체 전수조사(대시보드·리포트·랜딩·관리자단·파이프라인) + 이번 세션 실측 테스트
(베놈 자동 진단 케이스)를 원장의 실제 의사결정 질문에 대응시켜 격차를 도출했습니다.

## 1. 원장이 실제로 묻는 질문 7가지 → 제품 대응 현황

| # | 원장의 질문 | 현재 제품 | 판정 |
|---|---|---|---|
| 1 | "우리 병원, 지금 잘하고 있는 건가?" | 진단 점수 게이지 + 점수 구성 기여도 차트(신규) | ✅ 충족 |
| 2 | "옆 병원 대비 어디쯤인가?" | 벤치마크 표·리뷰 비교(병원 직접 입력값) | 🔶 부분 — 백분위("반경 내 상위 30%") 표현 필요 |
| 3 | "광고비가 아깝지 않은가?" | 미지원 | ❌ 격차 — 네이버 검색광고 API(광고주 계정)로 자사 광고 성과만 합법 연동 가능 |
| 4 | "그래서 당장 뭘 해야 하나?" | 우선 액션 3가지 + 근거 | 🔶 부분 — "직원에게 그대로 지시 가능한" 체크리스트 수준으로 구체화 여지 |
| 5 | "지난달보다 나아졌나?" | 추이 차트(2개월차부터) | ✅ 구조 충족 — 월간 재진단 운영만 필요 |
| 6 | "신환이 실제로 어디서 오나?" | 범위 밖 | ❌ 의도적 제외 — CRM/콜트래킹 영역. 리포트에 한계 고지 유지 |
| 7 | "우리 지역에서 어떤 키워드를 노려야 하나?" | **자동 키워드 산출(신규)** — 동/구/시/메인 × 진료과·시술 | ✅ 신규 충족 |

핵심 통찰: 원장의 언어는 "점수·순위"가 아니라 **"돈·환자·시간"**입니다. 모든 지표에는
"그래서 뭘 하면 되는지"가 붙어야 하고, 실제로 그 부분(액션)의 구체성이 재계약을 좌우합니다.

## 2. 이번에 반영한 것

- **점수 구성 기여도 차트** — "왜 62점인가"를 항목×가중치 분해로 시각화 (질문 1)
- **반경별 점수 비교 차트** — 생활권을 넓힐 때 경쟁 환경 변화 (질문 2 보조)
- **통합검색 커버리지 레이더 + 영역별 노출 막대** — 콘텐츠 영역 강약점 한눈에 (질문 7)
- **매트릭스 히트 셀** — 노출 위치가 상위일수록 진한 색 (가독성)
- **자동 키워드 산출 + 입지 분석 통합** (질문 7 + 개원의 니즈)
- **관리자단 상세분석** — 내부 전체판 열람 (컨설팅 준비 시간 단축)

## 3. 남은 격차와 우선순위 제안

**P1 (다음 스프린트)**
- 백분위 표현: "반경 1km 피부과 12곳 중 4위 수준" — 데이터는 이미 있음, 표현만 추가
- 액션의 지시문화: "플레이스 사진 최신화" → "대표사진 10장 교체: 외관1·데스크1·진료실2·장비2·대기실1·안내3"
- 리포트 마킹판/전체판 이중 산출 (회원가입 모델 결정 시)

**P2 (접수 서버 이후)**
- 자사 광고 성과 연동(검색광고 API, 광고주 동의 기반) — 질문 3 대응, 베놈 대행 고객에게 차별점
- 월간 자동 재진단 + 전월 대비 델타 자동 표기

**P3 (검토)**
- 플레이스 리뷰·사진 수 공개 집계값 자동 수집(법무 확인 후) — 현재는 병원 직접 입력
- 진료과 확장: 시술 사전 13과 → 세부 시술 확대는 상담 데이터로 보강

## 4. UI/UX 원칙 (전수조사에서 재확인)

- 원장의 열람 환경은 **모바일 + 30초**: 첫 화면에서 점수·등급·액션 1개가 보여야 함 (현재 충족)
- 전문용어 회피: "SERP·커버리지" 대신 "검색화면·노출 범위" — 화면 문구 전반 점검 완료
- 인쇄(PDF) 리포트가 원내 회의 자료로 쓰임: 인쇄 CSS 유지·개선
- 법무 고지는 각주가 아니라 지표 옆 문맥에: "미노출 = API 상위 5건 밖" 방식 유지

## 5. 성능 점검 결과

- 페이지: 단일 파일·외부 의존성 0 유지, 신규 차트 전부 인라인 SVG/CSS (추가 요청 0건)
- 파이프라인: 키워드 12개 자동 진단 = API 호출 약 84건, 3~4분 소요 — 배치 처리로 충분,
  실시간 서비스화 시에는 키워드당 캐시(24h)와 큐가 필요
- 어뷰징 제한(IP 일3회 등)이 자동 진단에도 그대로 적용되어야 함 (서버 구현 시)
