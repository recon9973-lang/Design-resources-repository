3. 데이터 구조 (v1과 동일 — 변경 없음)

수집은 공식 경로 3종만 사용합니다: ①건강보험심사평가원 병원정보서비스(공공데이터포털 공식 API — 병원명·주소·좌표·진료과), ②네이버 지역 검색 API(비로그인 공식 오픈 API, 키워드당 상위 5건, 월 1회, 일 한도 대비 극소량), ③통계청 SGIS(행정구역 인구·상권 지표). 화면 크롤링·실시간 추적·대량 수집·캡차 우회·로그인 수집은 하지 않으며, 리뷰 본문·환자정보는 수집하지 않습니다. 원본 API 응답은 병원에 노출하지 않고 내부 증빙으로만 접근통제(권한 분리 + 감사로그 + DB 트리거) 하에 보관합니다. 상세 명세는 v1 자료 5~7장과 동일하므로 첨부로 갈음합니다.

4. 배포·활용 시나리오 (법무 판단에 필요한 사실관계)

단계내용통제 장치
① 병원 접점 확보(a) 안내 페이지에서 병원이 직접 무료 진단 신청 (원장/담당자 성명·연락처·병원명 입력)
(b) 기존 거래처·소개 경로
(c) [질의 대상] 신규 병원에 대한 콜드 아웃리치(문자·메일) 가능 범위		신청 폼에 개인정보 수집·이용 동의 + 광고성 정보 수신 동의 체크박스
② 진단 리포트 생성·전달공식 API 데이터로 반경 경쟁 현황, 키워드 노출 등급, 인구 참고지표, 정보 정비 제안을 담은 PDF를 무료 제공표지·전면에 "(주)베놈 제공 무료 진단 · 내부 운영 참고용 · 환자 대상 광고물 전재 금지 · 순위/성과 보장 아님" 고지
③ 컨설팅 상담리포트를 근거로 개선 항목(플레이스 정보 정비, 키워드 커버리지 등)을 상담. 여기까지는 광고물 제작 없음상담 자료에도 동일 고지 · 경쟁 병원 평가성 표현 금지
④ 광고대행 계약 (별도)병원이 원하면 별도 광고대행 계약 체결. 광고물 제작·집행은 리포트와 분리된 기존 대행 프로세스로 진행의료광고 심의 대상 매체는 병원 명의로 심의 · 리포트 내용(경쟁 병원명·점수) 광고물 전재 금지 내규
당사가 인지하고 있는 경계선: 리포트가 ③에서 멈추면 "내부 진단자료"이지만, 리포트의 내용이 ④의 광고물에 그대로 옮겨지는 순간 의료광고 규제(비교·비방, 심의, 보장성 표현)가 적용된다는 점을 전제로 내규를 설계했습니다. 이 경계 설정이 충분한지가 이번 검토의 핵심 질의입니다.

6. 질의 사항

A. 무료 배포·대행사 모델 관련 (신규)

  1. 진단 리포트의 법적 성격 — 광고대행사가 병원에 무료로 제공하는 본 진단 리포트(환자 비노출, 내부 참고용 고지)가 의료법상 의료광고 또는 표시광고법상 광고에 해당할 가능성이 있는지. 무료·영업 목적이라는 점이 판단을 바꾸는지.
  2. 컨설팅·광고 제작의 책임 경계 — 리포트를 근거로 광고 컨설팅을 하고, 이후 별도 계약으로 광고물을 제작·집행할 때 (병원 명의 심의 전제) 리포트 단계와 광고물 단계의 책임을 분리하는 현재 내규(4장 ④)가 충분한지. 보완할 조치는.
  3. 병원의 무단 전재에 대한 당사 책임 — 병원이 고지를 무시하고 리포트 내용(경쟁 병원명, 점수)을 환자 대상 광고에 전재한 경우 당사의 방조·공동책임 가능성과, 면책을 위해 필요한 조치 수준(고지 문구, 제공 조건 동의서, 발견 시 조치 등).
  4. 리드젠 개인정보·광고성 정보 — 원장 연락처를 수집해 무료 리포트와 후속 상담 연락을 하는 구조에서, (a) 신청 폼 동의만으로 충분한지 (b) 병원 대표번호·공개 이메일로의 콜드 아웃리치가 허용되는 범위 (c) 필요한 동의 문구·기록 보관 요건.
  5. 무료 배포 시 필요한 최소 문서 세트 — 유료 약관 대신 무엇이 필요한지: 무료 자료 제공조건 고지(전재 금지·보장 없음·책임 한계), 개인정보 수집·이용 동의, 개인정보처리방침. 각각의 필수 기재 사항.

B. 데이터·표시 관련 (1차 메모 질의 유지 — 무료 모델에서도 동일한지 포함)

  1. 네이버 지역 검색 API 응답의 저장 기간·가공 표시 범위가 약관상 허용되는지 (무료 배포 자료에 쓰는 경우 포함).
  2. 월 1회 직원 수작업 검색화면 확인을 내부 증빙으로만 보관하는 것의 허용 여부 (기능은 승인 전 보류 중).
  3. 플레이스 리뷰 수·사진 수를 병원이 직접 입력하게 하는 현재 방식 외에, 당사 직원이 수기 확인해 입력하는 것의 법적 차이.
  4. 1인 의원 상호에 원장 성명이 포함된 경우의 처리 근거와 정정·삭제 요청 처리 기준.
  5. 공공데이터+API 결합 "진단 점수(당사 산식)"가 무료 자료에서도 표시광고·불공정거래 쟁점이 되는지, 필요한 한계 고지 수준.

7. 요청 결과물

  • 질의 항목별 허용 / 조건부 허용(조건 명시) / 금지 의견

  • 콜드 아웃리치 운영 기준 (가능하다면 체크리스트 형태)
