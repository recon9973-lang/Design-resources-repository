# 원본 SaaS 기획서 대조 · 갭 분석 · 보완 기획 (2026-07-05)

대상: `hospitallocalmarketingdashboardfulldesign.md`(2026-07-02 브레인스토밍 승인안)
방법: 기획서 항목 ↔ 현재 저장소 구축물 코드 레벨 전수 대조 → 갭 도출 → 보완 우선순위.
법무 정합(v2): 무상 배포 · 콜드콜 금지 · 리뷰 본문 미수집 · 보장/과장 표현 금지 — 아래 권장 전부 이 원칙 내.

## 1. 배경 — 기획서와 현 구축물의 관계

기획서는 **"반경 기반 병원 검색 노출 경쟁력 진단 SaaS"**(대시보드+월간 PDF+월간 스냅샷)를 정의한다.
그동안 실제로 구축된 것은 그 코어(스키마·점수엔진·API 명세)를 공유하되, **① 자기완결 진단 리포트 엔진
(`auto_diagnose`) ② 무료진단 리드 퍼널(landing→Supabase) ③ 회원앱(magic-link·리포트 게이팅) ④ 자가진단
3종·검색량·커버리지 레이더** 로 확장·선행되었다. 즉 **"영업·리드 전환 층"은 기획서보다 앞서 있고,
"인터랙티브 SaaS 대시보드 층"과 일부 측정 깊이는 아직 기획서에 못 미친다.**

## 2. 현황 대조표 (기획서 → 현재)

범례: ✅완료 · 🟡부분 · ⬜미착수

## 3. 갭 분석 — 무엇이 비었나 (우선순위)

### P1 · 제품 신뢰도·영업 직결 (지금 채우면 리포트 설득력↑)
1. **경쟁 병원 실측 벤치마크** (기획서 5.5) — 반경 내 상위 경쟁 병원 N곳을 골라, 같은 키워드 세트로 `content_exposure(kw, 경쟁사명)`를 돌려 **노출빈도·영역 커버리지**를 실측 → 레이더 벤치마크를 참고선에서 **실측선**으로 교체, 표로 제시. (실명 우열 단정 대신 "상위 경쟁군 평균" 익명 집계로 법무 안전화)
2. **플레이스 품질 15% 실측** — 리뷰수·평점·사진수·기본정보 완성도. 네이버 오픈 API 미제공 영역이라, **법무 검토 범위 내 수집 방식(공개 집계값만·본문 미수집)**을 정의하거나, 그전까지 **점수에서 제외/‘미측정’ 명시**해 목업 값이 종합점수를 왜곡하지 않게.
3. **종합 마케팅 경쟁력 0-100을 고객 리포트에 노출** — 4대 구성 합산 점수를 `auto_diagnose` 상단에 게이지+구성 분해로. (batch에만 있는 것을 고객 화면으로)

### P2 · SaaS 대시보드 층 (재방문·구독 전환)
4. **반경 500m 단위 선택 + `radius_metrics` 저장** — 현재 고정 1km. 500/1000/1500/2000 계산·비교.
5. **실제 지도 + 경쟁 병원 핀 레이어** — 실서버(지도 키) 구동 시. `staticmap` 준비됨.
6. **월간 추이("지난 리포트 대비 변화")** — `radius_metrics` 월별 저장 → 대시보드 변화 표시.
7. **원장 SaaS 대시보드 실연결** — 회원앱(로그인·프로필)과 대시보드 홈을 실데이터로 연결(현재 목업 폴백).

### P3 · 키워드·데이터 심화
8. **증상/고민·비교탐색형 키워드 확장** — 현재 동/구/시 × 진료과·시술만. `keyword_type` enum의 `symptom`·`comparison` 미생성 → 사전 추가(예: "어깨안올라감", "OO동 잘하는 곳").
9. **`action_recommendations` 구조화 저장** + `compliance_status` — 현재 리포트 내 텍스트 → 행 데이터로(추이·관리자 검수).
10. **PDF 리포트** — 자기완결 HTML → 인쇄/PDF(영업 배포용). 기획서 9.5 `reports/monthly`.
11. 상가정보·법정동코드 커넥터(주소 정규화·상권 보정) — 낮음.

## 4. 기획서 대비 "앞서 있는" 부분 (유지·강점)
자가진단 3종(리스크·비용·동선) · 검색량/연관검색어 · 커버리지 레이더+벤치마크 · 무료진단 리드 퍼널(동의 증적·어뷰징) · 회원앱(magic-link·마킹 해제) · 베놈 체크업 브랜딩. → **리드 전환 퍼널은 기획서보다 성숙.** 남은 과제는 "진단 깊이"와 "SaaS 대시보드 층".

## 5. 권장 로드맵 (2주 단위, 법무 정합 유지)
- **스프린트 A (제품 신뢰도)**: P1-1 경쟁 실측 벤치마크 → P1-3 종합 점수 노출 → P1-2 플레이스 품질 처리(수집 or 제외 명시).
- **스프린트 B (대시보드 층)**: P2-4 반경 선택+저장 → P2-7 회원앱↔대시보드 실연결 → P2-6 월간 추이. (실서버 배포 후 P2-5 지도)
- **스프린트 C (심화)**: P3-8 키워드 확장 → P3-9 액션 구조화 → P3-10 PDF.
- **보류(기획서와 동일)**: 월간 SERP 스냅샷은 법무 검토 완료 후 3단계에서.

## 6. 법무 정합 점검 (현행 유지 확인)
- 고위험 원천자료(스냅샷·HTML·payload)는 스키마상 `internal_evidence_items`(기본 `internal_admin_only`)로 분리 — 기획서 5.7·11.6과 일치. **고객 리포트엔 요약·등급·기준일만** 노출 원칙 유지.
- 경쟁 벤치마크는 **익명 집계("상위 경쟁군")**로 — 특정 병원 실명 우열 단정 금지(11.4).
- 플레이스 품질 수집은 **공개 집계값만·리뷰 본문 미수집**(11.4·11.5) — 수집 방식 확정 전엔 점수 제외.
- 액션은 운영/정보 정비 중심(치료효과·후기·전후·보장 표현 배제) — 현행 컨설팅 소견 준수.
