# 보안 점검 체크리스트 (준비 단계)

점검일: 2026-07-02 · 대상: hospital-marketing 전체 (정적 프로토타입 + 파이프라인)
상태 표기: ✅ 조치 완료 · ⏳ 서버/배포 단계에서 필요 · ⚠️ 운영 주의

## 1. 비밀정보 (Secrets)

| 항목 | 상태 | 내용 |
|---|---|---|
| API 키 저장 | ✅ | `.env`는 `.gitignore`로 커밋 차단. 저장소·클라이언트 코드에 키 없음 확인 |
| 키 로테이션 | ⚠️ | 개발 중 채팅으로 전달된 키는 정식 오픈 전 각 포털에서 재발급(로테이션) 권장 |
| 서버 환경변수 | ⏳ | 접수 서버 구축 시 키는 환경변수/시크릿 매니저로만 주입, 로그에 출력 금지 |

## 2. XSS / 입력 처리

| 항목 | 상태 | 내용 |
|---|---|---|
| 리포트 생성기 | ✅ | 병원명·키워드·API 응답 문자열(네이버 title 등 외부 입력) `html.escape` 적용 |
| 대시보드 | ✅ | `dashboard-live.json` 유래 문자열(병원명·주소·키워드) innerHTML 삽입 전 esc() 적용 |
| 입지 분석/관리자단 | ✅ | esc() 일괄 적용 |
| 신청폼 입력 | ⏳ | 서버에서 재검증(길이·형식·필수 동의), 저장 시 파라미터 바인딩(ORM/프리페어드) 필수 |

## 3. 페이지 보안 헤더

| 항목 | 상태 | 내용 |
|---|---|---|
| CSP | ✅ | 전 페이지 `<meta>` CSP: 외부 스크립트/프레임 차단, self+inline만 허용. GA4 도입 시 googletagmanager 도메인 추가 필요 |
| Referrer-Policy | ✅ | strict-origin-when-cross-origin (관리자단은 no-referrer) |
| 클릭재킹 | ✅ | frame-ancestors 'none' (서버 배포 시 X-Frame-Options 헤더로 보강) |
| HTTPS | ⏳ | GitHub Pages는 기본 HTTPS. 자체 도메인 연결 시 Enforce HTTPS 체크 |
| 서버 헤더 | ⏳ | 접수 서버에서 HSTS, X-Content-Type-Options:nosniff 등 응답 헤더로 승격 |

## 4. 크롤러/노출 통제

| 항목 | 상태 | 내용 |
|---|---|---|
| robots.txt | ✅ | admin/·location/·data/·pipeline/ 색인 차단 (landing만 공개 의도) |
| noindex 메타 | ✅ | 관리자단·대시보드·입지 분석·처리방침에 적용 |
| 1인 의원 노출 | ✅ | 검색 색인되는 공개 페이지에 병원명+점수 조합 미노출 (법무 B-9) |

## 5. 관리자단

| 항목 | 상태 | 내용 |
|---|---|---|
| 인증 | ⚠️ | **프로토타입은 인증이 없음 — 실데이터를 담아 공개 URL에 배포 금지.** 실서비스는 SSO/2FA 뒤에 배치 |
| 권한 분리 | ✅ | 스키마에 5단계 역할 + 법무 게이트 트리거 + 감사 로그 설계 완료 |
| 데모 데이터 | ✅ | 관리자단 목업의 IP·연락처는 마스킹된 가상 값만 사용 |

## 6. 어뷰징/가용성

| 항목 | 상태 | 내용 |
|---|---|---|
| 요청 제한 정책 | ✅ | IP/이메일/전화/병원 단위 제한 정의 (docs/어뷰징-방지-정책.md) + 스키마(diagnosis_requests, abuse_blocklist) |
| 클라이언트 가드 | ✅ | 허니팟, 3초 체류 검사, localStorage 일 2회 — UX용 (우회 가능 명시) |
| 서버 검증 | ⏳ | count_recent_requests() 기반 접수 전 검사, 인프라 rate limit(Cloudflare 등) |
| 외부 API 보호 | ✅ | 파이프라인 재시도 백오프, 월 1회 배치 — 호출량 자체가 극소 |
