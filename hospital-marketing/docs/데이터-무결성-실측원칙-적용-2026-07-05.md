# 데이터 무결성 — 실측만·미측정 명시·조작값 제거 (2026-07-05)

지시: 모든 자료는 실측만 사용. 자료 없는 항목은 "미측정"으로 명시. 환각·목업 절대 금지.
컨설팅 자료로서 법적 책임은 없으나 **신뢰도가 생명**.

## 1. 배경
리포트/파이프라인 전수 감사에서 **고객에게 도달하는 조작(하드코딩) 값**을 발견하여 전량 제거하고,
측정 불가 항목은 값을 지어내지 않고 "미측정"으로 표기하도록 원칙을 코드에 강제했다.

## 2. 발견된 조작값(제거 전) → 조치
| 위치 | 조작 내용 | 조치 |
|---|---|---|
| `auto_diagnose.py` `BENCH_DEFAULT` | 레이더 경쟁 기준선 78/55/66/22/58/48 (지어낸 값) | **완전 삭제.** 실측 벤치마크가 있을 때만 표시, 없으면 "경쟁 비교: 미측정" |
| `report.py` | `review_count 154·rating 4.4·photo_count 44` 기본값 | 삭제. 입력 없으면 `place=None`(미측정) |
| `export_dashboard.py` | `place_quality_score(154,4.4,44,0.7)` | `place=None`, 종합점수는 측정 축만 재정규화 |
| `validate_stage0.py` | 동일 | 동일 |
| `generate_report.py` | 동일 + 템플릿 `$s_place` | `place=None`, 화면 "미측정" 표기 |

감사 재확인: `grep place_quality_score(154 / BENCH_DEFAULT` → **잔존 0건.**

## 3. 강제한 원칙 — `scoring.composite_measured()`
종합 마케팅 경쟁력 점수는 **측정된 축만으로 가중 재정규화**한다. 미측정(None) 축은
임의값으로 채우지 않고 **제외**하며, 산정에 실제 반영된 가중 비중(`measured_weight_pct`)을
함께 반환해 리포트에 투명 고지한다.
- 노출 40 · 경쟁 25 · 수요 20 · **플레이스 15(공식 API 미제공 → 미측정)**
- 병원: 노출+경쟁+수요 = **반영 가중 85%** / 비병원(경쟁 데이터 없음): 노출+수요 = **60%**

## 4. 함께 구현한 실측 기능 (P1)
- **P1-3 종합 마케팅 경쟁력 점수** — 측정 축만으로 산정한 0-100 게이지 + 축별 분해(플레이스 = "미측정" 명시)를 고객 리포트 상단에 노출.
- **P1-1 실측 경쟁 벤치마크** — 반경 내 상위 경쟁 병원(HIRA 실명)을 **같은 키워드 세트로 공식 API 조회**해 영역별 노출률을 측정, 상위 경쟁군 평균을 레이더에 겹침(`--benchmark-competitors N`, 기본 3). 데이터가 없으면 표시하지 않고 "미측정". 법무: 실명 우열 단정 대신 **익명 집계("상위 경쟁 N곳 평균")**.

## 5. 각 지표의 데이터 출처·측정 상태 (현행)
| 지표 | 출처 | 상태 |
|---|---|---|
| 키워드 노출(매트릭스·레이더·막대) | 네이버 오픈 API(지역·6영역) | **실측** |
| 종합 노출 점수 | 위 노출 판정 집계 | **실측** |
| 경쟁 여유(밀도) | HIRA 반경 동일 진료과 | **실측**(병원만) |
| 수요·입지 | SGIS 인구가중 체감밀도 | **실측** |
| 검색량·연관검색어 | 네이버 검색광고 키워드도구 | 키 연동 시 실측, 없으면 표시 안 함 |
| 경쟁 벤치마크 | 경쟁사 공식 API 조회 | 측정 시 실측, 없으면 **미측정** |
| 실제 위치 지도 | 네이버 클라우드 Static Map | 키·개방망 시 실측, 없으면 링크/개략도 |
| 플레이스 **기본정보 완성도**(전화·링크·분류) | 네이버 지역 API | **자동 실측**(2026-07-05 구현) |
| 플레이스 **리뷰·평점·사진** | 공식 API 미제공 | 병원 직접 입력 시 **실측**, 미입력 시 그 항목만 미측정 |

## 6. 플레이스 품질 실측 구현 (2026-07-05, "미측정 → 측정")
공식 API가 리뷰/평점/사진을 주지 않으므로, 측정 가능한 것과 불가한 것을 분리해 각각 실측 경로를 마련.
- **기본정보 완성도(25%)**: 지역 API의 전화·링크·분류 존재 여부로 **자동 실측**. (예: 전화✕·링크○·분류○ → 67%)
- **리뷰수·평점·사진수(75%)**: 스크래핑은 법무 리스크라 배제. 대신 **병원이 자기 플레이스에서 확인해 입력**
  (`--reviews N --rating R --photos N`) → 입력 시 실측 반영. 미입력 시 그 세부항목만 미측정.
- `scoring.place_quality_partial()`: 측정된 세부항목만 재정규화하고 **측정 비중(frac)**을 반환.
  종합점수는 플레이스를 `15% × frac`만큼만 반영(기본정보만이면 place_frac 0.25 → 반영 가중 ~89%).
- 조작값 없음: 세부항목 하나도 지어내지 않음. 리포트에 "부분 25%"·"미입력"으로 투명 표기.

## 7. 남은 미측정 항목의 향후 실측 경로(값 조작 없이)
- 월간 스냅샷/실제 SERP: 기획서대로 법무 검토 후 3단계.
- 리뷰/평점/사진 자동화: 법무 검토 후 공개 집계값만 저빈도 수집(선택) — 그전까지 병원 입력이 유일한 실측.

## 8. 근거
코드: `medirank/scoring.py`(`composite_measured`), `auto_diagnose.py`(벤치마크 실측·종합점수·미측정 렌더), `report.py`·`export_dashboard.py`·`validate_stage0.py`·`generate_report.py`(하드코딩 제거). 재생성 리포트: auto-grand(종합 33.4·가중 85%)·auto-lames(39.8)·auto-venom(13.5·60%), 벤치마크 전부 미측정 표기.
