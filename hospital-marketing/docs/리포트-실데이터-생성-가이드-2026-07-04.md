# 리포트 실데이터 생성 가이드 — 검색량·연관검색어·실제 지도 (2026-07-04)

베놈 자체 서버/PC(개방망)에서 파이프라인을 돌려 **검색량·연관검색어·실제 지도**가
들어간 진단 리포트를 생성하는 절차. (Claude 웹 샌드박스는 `api.searchad.naver.com`·
지도 호스트가 차단돼 있어 이 환경에서는 생성 불가 — 반드시 개방망에서 실행.)

## 1. 필요한 키 — 어떤 기능에 무엇이 필요한가

| 기능 | 필요 키 | 발급처 | 비고 |
|---|---|---|---|
| 노출 진단(매트릭스) | `NAVER_CLIENT_ID/SECRET` | developers.naver.com 오픈 API(검색) | **이미 보유** — 재발급 불필요 |
| 인구·상권(입지) | `SGIS_CONSUMER_KEY/SECRET` | sgis.kostat.go.kr | 이미 보유 |
| 경쟁 병원 수 | `DATA_GO_KR_SERVICE_KEY` | data.go.kr 병원정보서비스 | 이미 보유 |
| **월 검색량·연관검색어** | `NAVER_AD_API_KEY` · `NAVER_AD_SECRET_KEY` · `NAVER_AD_CUSTOMER_ID` | **searchad.naver.com** → 도구 > API 사용 관리 | **신규 발급 필요**(검색광고=별도 서비스) |
| **실제 위치 지도** | `NAVER_MAP_KEY_ID` · `NAVER_MAP_KEY` | **console.ncloud.com** → AI·NAVER API > Maps | 신규 발급(선택) |

> ⚠️ 흔한 오해: 지금 가진 네이버 **오픈 API 키(openapi.naver.com)** 로는 검색량·연관검색어가
> **안 나온다.** 그건 오직 네이버 **검색광고 API(api.searchad.naver.com)** 에서만 제공된다.
> 두 API는 계정·인증·호스트가 완전히 다르다. 오픈 API는 그대로 두고, 검색광고 키만 새로 발급.

## 2. 검색광고 키 발급 (월 검색량·연관검색어)

1. searchad.naver.com 로그인(사업자/개인 광고주 계정, **광고 집행·과금 없이 키만 발급 가능**)
2. 우측 상단 **도구 > API 사용 관리**
3. **네이버 검색광고 API 라이선스 발급** → `API_KEY`(액세스 라이선스)·`SECRET_KEY`(비밀키) 확인
4. **CUSTOMER_ID** = 내 계정 번호(우측 상단 계정 정보의 숫자 ID)
5. 서명 방식: `HMAC-SHA256( SECRET, "{ms타임스탬프}.GET./keywordstool" )` → 코드에 이미 구현됨

## 3. 실행

```bash
cd hospital-marketing/pipeline
cp .env.example .env          # 최초 1회
# .env에 위 5~7개 키를 채운다 (특히 NAVER_AD_*, 선택적으로 NAVER_MAP_*)

# 업체명으로 진단 → JSON + HTML 리포트 생성
python3 auto_diagnose.py "그랜드아름다운의원" --region 춘천 \
    --out ../data/auto-grand.json \
    --html ../reports/auto-grand.html \
    --html-masked ../reports/auto-grand-masked.html
```

- 키가 있으면 리포트에 **`월 검색량` 열 + `검색 수요·연관 검색어` 섹션 + 실제 지도 이미지**가 자동 표시.
- 키가 없으면 해당 요소만 조용히 생략(기존 리포트와 동일) — 나머지는 정상 생성.
- 마킹판(`--html-masked`)에는 검색량·실지도 등 민감 수치가 **데이터 단계에서 제외**됨.

## 4. 동작 확인 포인트

- 콘솔 로그에 `[검색 수요] … 검색량 확보 N개 · 연관검색어 M개`, `[실제 지도] … 임베드 완료`가 뜨면 성공.
- HTML에서 `검색 수요 · 연관 검색어` 섹션과 `.realmap` 이미지가 보이는지 확인.
- 검색광고 호출이 401/403이면: CUSTOMER_ID·서명 시각(서버 시계) 확인. 지도 403이면 NCP Maps 이용신청 상태 확인.
