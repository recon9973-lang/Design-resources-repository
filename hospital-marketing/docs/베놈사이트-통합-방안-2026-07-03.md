  상담 전화 1661-4142, 공지 게시판(`/board/`) 운영. 자체 도메인·자체(혹은 빌더) 웹사이트.
- **보유 채널**: blog.naver.com/venomad(구독 60+), tv.naver.com/venomad(영상 10+) — "대구병원마케팅" 웹문서 1위

## 1. 통합 옵션 비교

| | A. 서브도메인 (권장) | B. 사이트 내 페이지 | C. 현행 유지 + 링크 |
|---|---|---|---|
| 형태 | `diagnosis.venomad.com` | `venomad.com/diagnosis` | Pages 링크를 메뉴/배너로 연결 |
| 브랜드 신뢰 | 높음 (자사 도메인) | 가장 높음 | 낮음 (github.io 노출) |
| SEO 효과 | 본 도메인과 분리 축적 | 본 도메인에 직접 축적 | 없음 |
| 기술 부담 | 낮음 — DNS + 정적 호스팅이면 즉시 | 중간 — 기존 사이트 빌더/스택에 이식 필요 | 없음 |
| 배포 독립성 | 완전 독립 (지금 워크플로 그대로) | 사이트 배포에 종속 | 완전 독립 |
| 접수 서버 확장 | 서브도메인에 API 서버 추가 용이 | 사이트 스택에 종속 | 별도 구축 |

**권장: A안.** 이유 — ① 진단 도구는 단일 파일 정적 구조라 어떤 정적 호스팅에도 그대로 올라감
② 기존 사이트 스택(빌더 여부)을 건드리지 않아 리스크 0 ③ 이후 접수 서버·회원 기능을
서브도메인 안에서 독립적으로 키울 수 있음 ④ B안은 사이트 리뉴얼 시점에 흡수하면 됨.

## 2. 퍼널 설계 (회원가입-마킹 모델과 결합)

```
venomad.com (본 사이트)                diagnosis.venomad.com (진단)
  메뉴 "무료 검색정보 진단" ───────▶  랜딩(신청폼: 필수 동의만으로 신청 가능)
  블로그/네이버TV 콘텐츠 CTA ──────▶      │  접수 서버: 동의 증적(일시·IP·문구버전) 저장
                                          ▼
                                     auto_diagnose 배치 실행 (업체명→키워드 자동 산출)
                                          │
                              ┌───────────┴───────────┐
                              ▼                       ▼
                        마킹판 리포트(즉시 발송)   전체판(무료 회원가입 후 열람)
                              │                       │
                              └──────── 상담 동의자만 → 베놈 컨설팅 → 광고대행 계약
```

- 두 단계 모두 무료 → v2 법무 검토의 대전제(무상 배포) 유지
- 광고성 수신 동의는 가입 조건으로 강제하지 않음 (선택 동의 유지 — 필수)
- 관리자단 상세분석 = 내부 전체판 (이미 구현) → 상담 준비 자료로 직결

## 3. 기술 체크리스트 (A안 기준)

1. DNS: `diagnosis` CNAME → 호스팅(현 GitHub Pages도 커스텀 도메인 지원 — Settings→Pages→Custom domain)
2. 정적 이전은 파일 복사로 끝 (전 페이지 단일 파일·상대 경로) — canonical/OG URL만 도메인으로 치환
3. GA4: 크로스 도메인 추적 설정(본사이트↔서브도메인) 또는 별도 속성 — 유입 채널 구분 목적이면 별도 속성 권장
4. 접수 서버(2단계): 서브도메인 하위 `/api` — 폼 제출, 동의 증적, 어뷰징 카운터(IP 일3회 등),
   suppression 체크. DB 스키마는 `db/schema.sql` 그대로 사용 가능
5. 회원(3단계): 이메일 링크 로그인(비밀번호 없는 매직링크)이 병원 담당자 UX에 적합 + 개인정보 최소화

## 4. 법무·운영 유의점

- 개인정보 처리 주체가 (주)베놈으로 일원화되므로 처리방침 1부로 통합 가능 — placeholder 13개 기입 후 게시
- 도메인 이동 시에도 필수 고지 문구(내부 참고용·전재 금지·당사 산식·보장 없음) 그대로 유지
- 본 사이트에서 진단으로 보내는 배너 문구도 의료광고 가이드 적용: "순위 보장" 류 표현 금지,
  "검색정보 운영 상태를 무료로 진단해 드립니다" 수준 권장
- github.io 링크는 이전 완료 후 서브도메인으로 301 안내 (Pages는 커스텀 도메인 설정 시 자동 리다이렉트)
