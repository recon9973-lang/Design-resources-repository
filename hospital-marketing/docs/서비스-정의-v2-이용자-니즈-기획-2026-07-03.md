# 서비스 정의 v2 — 정책·구조·이용자 니즈 기획 (2026-07-03)

리서치·브레인스토밍 결과를 바탕으로 프로그램의 정책과 구조를 재정의합니다.
계기: 관리자단 "증빙 검수" 화면이 이해되지 않는다는 대표님 피드백 — 근본 원인은
**"누구를 위한 화면인가"가 정의되지 않은 채 기능이 쌓인 것**이었습니다.

---

## 1. 정책 선언 (운영 주체 관점)

| 항목 | 정의 |
|---|---|
| 운영 주체 | (주)베놈 — 병원 전문 광고대행사 |
| 이 툴의 목적 | **수익 창출 없음.** 회사 이미지 제고 + 서비스의 일환 (기술력 증명, 신뢰 형성) |
| 수익과의 관계 | 툴 자체는 끝까지 무료. 광고대행 계약은 이용자가 원할 때만 이어지는 **부산물** |
| 법무 프레임 | 무상 배포 전제(법무 v2) 유지 — 판매·유료화 없음이 명문화되어 있어 그대로 유효 |
| 브랜드 톤 | "우리가 쓰는 도구를 그대로 공개합니다" — 영업 문서가 아니라 **공개 도구** |

이미지 제고형 무료 도구의 성공 공식은 "정확함 + 아낌없음 + 팔지 않음"입니다.
마킹판도 이 정책과 충돌하지 않게: 마킹은 "판매 유도"가 아니라 "회원 전환" 목적이며
전체판도 무료임을 항상 병기합니다.

## 2. 구조 재정의 — 화면은 두 세계로 나뉜다

혼란의 원인: 이용자용 화면과 베놈 내부용 화면이 같은 급으로 존재하는데 구분 표기가 약했습니다.

```
[이용자 세계 — 공개]                    [베놈 내부 세계 — 비공개]
 랜딩(신청)                              관리자단
 진단 대시보드                            ├ 신청 관리 (리드 퍼널)
 진단 리포트 (마킹판/전체판)               ├ 상세 분석 (전체판 열람)
 입지 분석                                ├ 어뷰징 모니터
                                         ├ 수신거부 목록
                                         ├ 증빙 검수  ← 문제의 화면
                                         └ 방문 통계
```

### "증빙 검수"란 무엇인가 (쉬운 설명)

이용자에게 "노출/미노출"이라고 말하려면 **그 근거를 남겨야** 합니다(광고 결과 분쟁 대비).
증빙 검수는 그 근거 자료(API 원본 응답 등)의 **내부 보관함**입니다. 각 행 = 근거 1건이고,
"법무 상태"는 그 근거를 고객 자료에 써도 되는지의 결재 상태입니다.

다만 현재 화면엔 **아직 만들지 않기로 한 기능(검색화면 스냅샷·광고영역 관찰)의 예시 데이터**가
섞여 있어 혼란을 만들었습니다. 이 유형들은 법무 보류로 수집 자체를 안 하므로 예시에서도 뺍니다.

**개편안 (즉시 적용)**: ① 화면 상단에 한 줄 정의 추가 ② 보류 기능의 목업 행 제거 ③ 용어를
한국어로 (raw_api_payload → "API 원본 응답") ④ 탭명 "증빙 검수" → "근거 자료함".

## 3. 이용자 페르소나 4종과 궁금증 → 기능 매핑

리서치 근거: 원장 관심사는 플레이스·리뷰·대행사 검증에 집중, 소상공인 대상 무료
자가진단 도구는 시장 공백, 마케터는 데이터랩·블랙키위류 키워드 도구를 이미 사용.

### P1. 병원 원장 (핵심 페르소나)
| 궁금증 | 현재 | 대응 |
|---|---|---|
| "환자들이 검색하면 우리 병원이 나오나?" | ✅ 키워드 노출 진단 | 유지 |
| "옆 병원과 비교하면?" | 🔶 벤치마크 | 백분위 표현 추가 (P1 백로그) |
| "리뷰·별점 관리는 잘 되고 있나?" | 🔶 직접 입력 | 리뷰 응대 체크리스트 콘텐츠 추가 |
| "대행사가 일을 제대로 하고 있나?" | ❌ | **대행사 점검 모드** (신규 — 아래) |
| "광고비 대비 효과는?" | ❌ | 자사 광고 성과 연동 (광고주 동의, P2) |

### P2. 소상공인 (확장 페르소나)
| 궁금증 | 현재 | 대응 |
|---|---|---|
| "내 가게, 네이버에서 어떻게 보이나?" | ✅ 자동 진단이 이미 비병원 지원 | **입력 문턱 낮추기**: 업체명 하나 → 셀프 진단 |
| "뭐부터 고쳐야 하나?" | 🔶 병원 중심 액션 | 업종 중립 액션 사전 추가 |
| "돈 안 들이고 할 수 있는 게 뭔가?" | ❌ | "0원 마케팅 체크리스트" 콘텐츠 |

### P3. 마케터 (인하우스/프리랜서)
| 궁금증 | 현재 | 대응 |
|---|---|---|
| "이 키워드 영역 구성이 어떻게 되나?" | ✅ 6영역 매트릭스 | 유지 — 마케터에게 이미 유용 |
| "보고서에 바로 쓸 수 있나?" | 🔶 PDF 인쇄 | 리포트 공유 링크(현재 방식) 정식화 |
| "데이터 출처가 명확한가?" | ✅ 전 화면 고지 | 유지 — 차별점으로 강조 |

### P4. 자사 점검 (베놈 스스로 / 대행사 검토 중인 사람)
| 궁금증 | 현재 | 대응 |
|---|---|---|
| "베놈은 자기 마케팅은 잘하나?" | ✅ 셀프 진단 공개 (auto-venom) | **"우리도 이 도구로 우리를 진단합니다" 페이지로 정식화** — 신뢰 장치 |
| "이 대행사 믿어도 되나?" | 🔶 | 진단 리포트 자체가 증명 — 산식·출처 공개 유지 |

## 4. 브레인스토밍 백로그 (효과 × 난이도)

**A. 대행사 점검 모드** (원장 니즈 1순위, 베놈 이미지와 정합) — 지금 대행사가 있는 병원이
자기 병원을 진단해 "대행 성과를 스스로 검증"하는 프레임. 기능은 동일, 문구·리포트 관점만 추가.
난이도 낮음 / 효과 큼. **베놈에 오히려 유리**: 잘하는 대행사만 이 프레임을 환영할 수 있음.

**B. 셀프 진단 웹 실행** — 지금은 베놈이 CLI로 돌려주는 구조. 이용자가 랜딩에서 업체명 입력
→ 서버가 auto_diagnose 실행 → 마킹판 즉시 표시. 접수 서버 결정에 걸림. 난이도 중 / 효과 최대.

**C. 월간 변화 리포트** — "지난달보다 나아졌나"는 모든 페르소나 공통 질문. 재진단 자동화 + 델타
표시. 난이도 중 / 효과 큼 (재방문 동기 = 이미지 제고 목적에 최적).

**D. 0원 마케팅 체크리스트** — 소상공인용 콘텐츠 (플레이스 정보 완성도, 리뷰 응대, 사진 구성).
진단 결과와 연결된 실행 가이드. 난이도 낮음 / 효과 중.

**E. 키워드 영역 구성 뷰어** — 마케터용: 키워드 하나 넣으면 6영역 활성/구성만 빠르게 보여주는
경량 도구. 난이도 낮음 (기존 커넥터 재사용) / 효과 중 (마케터 입소문 채널).

**F. 자사 광고 성과 연동** — 검색광고 API, 광고주 동의 기반. 난이도 높음 / 효과 큼 (P2 유지).

## 5. 우선순위 제안

1. **즉시**: 증빙 검수 화면 개편(§2) + 대행사 점검 모드 문구(A) + 자사 셀프 진단 페이지 정식화(P4)
2. **접수 서버 결정 후**: 셀프 진단 웹 실행(B) → 월간 변화 리포트(C)
3. **병행 콘텐츠**: 0원 체크리스트(D), 키워드 영역 뷰어(E)
4. **후순위**: 광고 성과 연동(F)
