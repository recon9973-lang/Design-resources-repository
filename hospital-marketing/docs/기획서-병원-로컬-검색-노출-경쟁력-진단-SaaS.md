# 병원 로컬 검색 노출 경쟁력 진단 SaaS 기획서

작성일: 2026-07-02  
상태: 브레인스토밍 승인안  
작업명: 병원 원장용 반경 기반 마케팅 경쟁력 대시보드

## 1. 제품 한 줄 정의

운영 중인 병원 원장이 내 병원 주소와 진료과를 입력하면, 반경 500m 단위 안의 경쟁 병원 수, 인구/상권 수요, 네이버 검색 노출 상태를 종합해 월간 마케팅 경쟁력을 진단하고 개선 액션을 제안하는 SaaS 대시보드.

## 2. 확정된 방향

- 제품 전략: 혼합형
  - MVP는 기존 병원의 월간 마케팅 경쟁력 진단 SaaS
  - 확장 상품은 개원 입지 분석 프리미엄 리포트
  - 장기 확장은 대행사/컨설턴트용 멀티 병원 관리 플랫폼
- 첫 고객: 이미 운영 중인 병원 원장
- 첫 진료과: 고경쟁 비급여 중심
  - 피부과, 성형외과, 치과, 한의원, 안과, 비만/다이어트, 탈모 등
- 검색 측정: 네이버 중심
- 데이터 방식: 공공데이터/API 기반 + 네이버 노출 지표 별도 수집
- 리스크 조정안: B-safe
  - SaaS 대시보드 + 월간 PDF 리포트
  - 공식 API 중심
  - 실제 검색화면 스냅샷은 법무 검토 후 저빈도/월간 검증 용도로 제한
  - 위험도가 높은 원천자료는 고객 화면에 직접 노출하지 않고 관리자단 내부 검수/증빙 자료로만 관리

## 3. 제품명 후보

1. 메디랭크 로컬
2. 병원 상권 랭크
3. 메디플레이스 인사이트
4. 닥터랭크 맵
5. 우리병원 노출진단

MVP 영업 문구는 브랜드명보다 문제를 바로 말하는 것이 좋다.

권장 문구:

- "우리 병원, 주변 1km 안에서 얼마나 잘 보이고 있나요?"
- "공공데이터와 공식 API 기반 병원 검색 노출 경쟁력 진단"
- "월간 병원 검색 노출 경쟁력 리포트"

피해야 할 문구:

- "네이버 실시간 순위 추적"
- "플레이스/파워링크 완전 자동 수집"
- "경쟁 병원 광고 데이터 완전 분석"
- "상위노출 보장"

## 4. 사용자 시나리오

### 4.1 운영 중 병원 원장

1. 병원명, 주소, 진료과를 입력한다.
2. 반경을 500m, 1km, 1.5km, 2km 단위로 선택한다.
3. 내 병원 주변 경쟁 병원 수와 위치를 지도에서 본다.
4. 핵심 키워드별 네이버 공식 API 기준 노출 상태를 확인한다.
5. 월간 검증 스냅샷이 있는 키워드는 실제 검색화면 기준 참고 순위를 확인한다.
6. 반경 내 경쟁 병원 대비 마케팅 경쟁력 점수를 본다.
7. 이번 달 우선 개선 액션 3가지를 확인한다.
8. PDF 리포트로 저장하거나 내부 직원/마케팅 대행사에 공유한다.

### 4.2 마케팅 담당자 또는 대행사

1. 여러 병원을 등록한다.
2. 병원별 월간 노출 점수와 경쟁 강도를 비교한다.
3. 개선 액션을 제안서나 월간 보고서에 붙인다.
4. 장기적으로 고객 병원별 변화 추이를 관리한다.

### 4.3 개원 준비 원장

MVP 이후 확장한다.

1. 후보 입지 주소를 여러 개 입력한다.
2. 각 후보지의 반경별 경쟁 병원 수와 인구/상권 수요를 비교한다.
3. 진료과별 공백 지역과 과밀 지역을 확인한다.
4. 프리미엄 PDF 리포트 또는 컨설팅 상품으로 판매한다.

## 5. MVP 핵심 화면

### 5.1 병원 등록/검색 화면

입력값:

- 병원명
- 주소
- 대표 진료과
- 세부 키워드 묶음
- 네이버 플레이스 URL 또는 사업장 식별 정보

출력:

- 병원 위치 좌표
- 공공데이터 병원 정보 매칭 결과
- 네이버 지역 API 검색 매칭 후보
- 사용자가 확인한 내 병원 식별값

### 5.2 대시보드 홈

구성:

- 마케팅 경쟁력 점수 0-100
- 반경 선택 컨트롤: 500m, 1km, 1.5km, 2km
- 경쟁 병원 수
- 핵심 키워드 평균 노출 상태
- 미노출/취약 키워드 수
- 지난 리포트 대비 변화
- 이번 달 액션 3가지

핵심 원칙:

원장은 분석가가 아니므로 첫 화면은 "지도 -> 점수 -> 취약 키워드 -> 액션" 순서로 보여준다.

### 5.3 지도 분석 화면

지도 레이어:

- 내 병원 중심점
- 반경 링
- 같은 진료과 경쟁 병원 점
- 유사 키워드 경쟁 병원 점
- 인구 밀집도 또는 행정동/집계구 수요 지표
- 상권/업종 보조 지표

필터:

- 반경
- 진료과
- 세부 키워드
- 경쟁 병원 노출 등급
- 신규/폐업/운영 상태

### 5.4 키워드 노출 화면

키워드 단위 표시:

- 키워드명
- 카테고리: 대표 키워드, 시술 키워드, 지역 키워드, 고민 키워드
- 공식 API 기준 노출 여부
- API 결과 내 위치
- 월간 검증 스냅샷 상태
- 스냅샷 기준 참고 순위
- 상위 경쟁 병원
- 내 병원의 격차
- 추천 액션

예시:

| 키워드 | API 기준 | 스냅샷 | 경쟁 상위 병원 | 진단 |
| --- | --- | --- | --- | --- |
| 강남 피부과 | 노출 | 참고 11위 | A피부과, B의원 | 상위권 밖 |
| 강남 리프팅 | 노출 | 참고 7위 | C의원 | 개선 가능 |
| 강남 탈모치료 | 미노출 | 미검증 | D의원 | 키워드 커버리지 필요 |

### 5.5 경쟁 병원 벤치마크

비교 항목:

- 병원명
- 거리
- 진료과/카테고리
- 공식 API 노출 빈도
- 핵심 키워드 커버리지
- 리뷰 수
- 평점
- 사진 수
- 주소/카테고리 일치도
- 월간 검증 스냅샷 기준 상위권 여부

주의:

- 리뷰 본문, 환자 치료경험담, 의료효과 표현은 MVP에서 분석하지 않는다.
- 리뷰 수/평점 같은 공개 집계값만 제한적으로 사용한다.

### 5.6 액션 리포트

액션은 의료광고 위반 위험이 없도록 운영 개선/콘텐츠 정비 중심으로 제안한다.

허용 예시:

- 네이버 플레이스 진료 카테고리와 설명 문구를 실제 진료 항목과 맞춘다.
- 대표 사진, 진료시간, 주차, 예약 링크 등 기본 정보를 최신화한다.
- 미노출 키워드는 병원 소개/진료 항목/FAQ 구조를 점검한다.
- 경쟁 병원 대비 리뷰 수가 부족한 경우 합법적인 만족도 조사와 재방문 안내 프로세스를 정비한다.
- 지역 키워드와 진료 키워드 조합을 월간 리포트에서 추적한다.

금지 또는 법무 검토 필요:

- 치료 효과 보장 표현
- 환자 후기/치료경험담을 광고 소재로 자동 추천
- 전후사진을 통한 성과 강조
- 특정 병원 비방 또는 비교우위 단정
- 상위노출 보장

### 5.7 관리자단 내부 검수 화면

고객에게 직접 보여주기 위험한 자료는 관리자단에서만 확인한다. 고객 화면에는 가공된 점수, 등급, 기준일, 출처, 요약 진단만 노출한다.

관리자단에서만 볼 자료:

- 실제 검색화면 스냅샷 이미지
- 검색 결과 원문 HTML 또는 구조화 전 원천 데이터
- 파워링크/플레이스/통합검색 섹션의 원천 관찰값
- 수집 실패, 차단, 캡차 발생, 비정상 응답 로그
- 네이버 API 원본 payload
- 병원 매칭 신뢰도와 수동 검수 이력
- 경쟁 병원별 상세 원천 증빙
- 법무 검토 상태와 위험 플래그
- 수집 기준 위치, 시간, 기기, 로그인 여부 등 운영 메타데이터

고객 화면에 보여줄 수 있는 자료:

- 공식 API 기준 노출 여부
- 월간 참고 스냅샷 존재 여부
- 키워드별 상위권/중위권/미노출 같은 등급화된 결과
- 반경 내 경쟁 병원 수
- 경쟁력 점수와 변화 추이
- 출처, 기준일, 측정 조건 요약
- 의료광고 리스크를 제거한 개선 액션

고객 화면에 직접 노출하지 않을 자료:

- 검색화면 원본 이미지 전체
- 원문 HTML
- 크롤링/수집 실패 로그
- 차단/캡차 관련 정보
- 경쟁 병원 광고 문구 원문
- 리뷰 본문 원문
- 법무 리스크 내부 메모

## 6. 점수 체계

최종 점수: 마케팅 경쟁력 0-100점

### 6.1 노출 점수 40%

측정 항목:

- 핵심 키워드별 공식 API 기준 노출 여부
- API 결과 내 상대 위치
- 상위 3/5/10위권 진입 여부
- 미노출 키워드 비율
- 월간 검증 스냅샷 참고 순위

주의:

- 실제 SERP 순위와 API 결과는 다를 수 있으므로 분리 표기한다.
- 고객에게 "실시간 순위"라고 표현하지 않는다.

### 6.2 경쟁 밀도 점수 25%

측정 항목:

- 반경 500m, 1km, 1.5km, 2km 내 같은 진료과 병원 수
- 유사 키워드 병원 수
- 상위 노출 경쟁 병원 수
- 경쟁 병원 거리 가중치

산식 예시:

- 가까운 병원일수록 가중치 높음
- 같은 진료과는 1.0
- 유사 진료과는 0.5-0.8
- 같은 키워드 상위 노출 병원은 추가 가중치

### 6.3 수요/입지 점수 20%

측정 항목:

- 반경 내 인구 밀집도
- 타깃 연령대 비중
- 행정동/집계구 기반 생활권 지표
- 상권 보조 지표

주의:

- 인구 데이터는 위치 기반 추정이므로 정확한 방문 수요로 단정하지 않는다.
- "잠재 수요 참고지표"로 표기한다.

### 6.4 플레이스 품질 점수 15%

측정 항목:

- 리뷰 수
- 평점
- 사진 수
- 영업시간/예약/전화/주차 등 기본 정보 완성도
- 카테고리/진료 키워드 커버리지

주의:

- 리뷰 본문 분석은 MVP 제외
- 환자 후기 활용 추천은 의료광고 리스크가 있어 제외

## 7. 데이터 소스

### 7.1 병원 기본정보

주요 소스:

- 건강보험심사평가원 병원정보서비스
- 공공데이터포털 제공
- 무료 API
- REST/XML
- 병원 주소, 진료과, 기관 정보 기반

활용:

- 병원 마스터 생성
- 내 병원 공공정보 매칭
- 경쟁 병원 후보 생성

### 7.2 상권/의료기관 보조 데이터

주요 소스:

- 소상공인시장진흥공단 상가(상권)정보
- CSV 파일 데이터
- 분기 업데이트
- 상호명, 업종코드, 주소, 좌표 포함

활용:

- 병원/의료기관 보조 매칭
- 상권 밀도 분석
- 좌표 보정

### 7.3 행정/지역 코드

주요 소스:

- 행정안전부 법정동코드 API

활용:

- 주소 정규화
- 법정동/행정동 매핑
- 공공데이터 조인 키 생성

### 7.4 인구/공간 데이터

주요 소스:

- SGIS 통계지리정보서비스
- 집계구/행정동 경계
- 인구 관련 지표

활용:

- 반경 내 인구 밀집도
- 지역 수요 참고지표
- 개원 입지 분석 확장

### 7.5 네이버 공식 API

주요 소스:

- 네이버 지역 검색 API
- 네이버 검색 API 계열
- 비로그인 방식 오픈 API

활용:

- 업체 노출 여부
- API 결과 내 상대 위치
- 주소/좌표 기반 병원 매칭
- 공식 API 기준 지표 생성

제한:

- 실제 플레이스/파워링크/통합검색 화면 전체를 대체하지 않는다.
- 호출 한도와 표시 정책을 지킨다.

### 7.6 월간 검증 스냅샷

상태:

- 법무 검토 후 제한 적용

운영 방식:

- 병원당 핵심 키워드 10-30개
- 월 1회
- 기준 위치, 시간, 기기, 로그인 여부 기록
- 실제 화면 순위는 "참고 스냅샷"으로 표기

## 15. 핵심 차별점

1. 병원 원장이 이해하는 언어로 말한다.
   - "API 순위"가 아니라 "우리 병원이 주변 경쟁 병원 대비 얼마나 보이는가"
2. 반경 기반이다.
   - 병원 마케팅은 전국 순위보다 생활권 경쟁이 중요하다.
3. 검색 노출과 입지를 합친다.
   - 검색에서 안 보이는 문제와 주변 경쟁이 과밀한 문제를 함께 진단한다.
4. 액션까지 준다.
   - 단순 리포트가 아니라 이번 달 개선 우선순위를 제안한다.
5. 법무 리스크를 낮춘다.
   - 공식 API와 공공데이터 중심으로 설계하고, 실제 검색화면 스냅샷은 제한적으로 다룬다.

## 16. 오픈 리스크

1. 네이버 API 결과와 실제 검색 화면 결과의 차이
   - 대응: 공식 API 기준 지표와 월간 참고 스냅샷을 분리 표기
2. 병원 데이터 매칭 오류
   - 대응: 사용자가 내 병원을 확인/수정하는 검수 단계 제공
3. 진료과/키워드 분류 난이도
   - 대응: 고경쟁 비급여 과부터 시작하고 키워드 사전을 수동 관리
4. 의료광고 리스크
   - 대응: 추천 액션을 운영/정보 정비 중심으로 제한
5. 대량 수집 리스크
   - 대응: MVP에서 실시간/대량 수집 제외
6. 데이터 갱신 비용
   - 대응: 월간 리포트 모델로 시작해 수집 빈도 제한
7. 고위험 원천자료 노출 리스크
   - 대응: 관리자단 전용 보관, 고객용 안전 요약값 분리, 권한/감사 로그 적용

## 17. 최종 권장안

MVP는 "병원 원장용 월간 검색 노출 경쟁력 진단 SaaS"로 출시한다.

제품은 SaaS 대시보드 형태지만, 초반 영업에서는 PDF 리포트가 강한 설득 도구가 된다. 따라서 대시보드와 리포트를 동시에 설계하되, 데이터 수집은 공식 API와 공공데이터 중심으로 제한한다.

실제 검색화면 순위는 제품의 핵심 약속으로 삼지 않는다. 법무 검토 전에는 "월간 참고 스냅샷"으로만 운영하고, 실시간 순위 추적이나 대량 자동 수집은 MVP에서 제외한다.

위험도가 높은 자료는 외부 상품 화면이 아니라 내부 자료로만 사용한다. 원본 스냅샷, 원문 HTML, 수집 로그, 광고 섹션 관찰값, 법무 메모는 관리자단에서만 확인하고, 고객에게는 검토된 요약 지표와 안전한 액션만 제공한다.

첫 유료 고객에게 팔 문장은 다음이 가장 안전하고 강하다.

"우리 병원이 주변 1km 안에서 경쟁 병원 대비 얼마나 잘 검색되고 있는지, 공공데이터와 공식 API 기반으로 매달 진단해드립니다."
