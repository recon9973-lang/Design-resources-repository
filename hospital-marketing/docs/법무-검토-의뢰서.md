## 1. 서비스 개요

병원 원장 대상 SaaS. 병원 주소·진료과를 입력하면 다음을 월간 리포트로 제공:

- 반경 500m~2km 내 경쟁 병원 수·분포 (건강보험심사평가원 공공데이터)
- 핵심 키워드의 네이버 **공식 오픈 API** 기준 노출 여부 (비로그인, 일 한도 내)
- 지역 인구·상권 참고지표 (통계청 SGIS)
- 운영·정보 정비 중심의 개선 액션 제안

수익모델: 월 구독 (9만~79만원대), 리포트 PDF 제공.

## 2. 검토 요청 사항

### 2.1 (핵심) 네이버 검색화면 월간 스냅샷 수집의 적법성

현재 MVP는 공식 API만 사용하며 검색화면 수집은 **하지 않습니다**.
차기 단계에서 아래 방식의 "월간 참고 스냅샷"을 검토 중입니다:

- 병원당 키워드 10~30개, **월 1회**, 사람이 보는 수준의 저빈도 접근
- 비로그인, 기준 위치·시간·기기 조건을 기록
- 캡차/차단 우회 없음, 대량·실시간 수집 없음
- 원본 화면은 내부 증빙으로만 보관, 고객에게는 등급화된 요약만 표시

**질의:**
1. 위 방식이 네이버 이용약관·robots 정책·부정경쟁방지법(성과물 무단사용)·
   저작권법 관점에서 허용되는 범위인지, 허용된다면 지켜야 할 조건은?
2. 수작업(직원이 직접 검색 후 화면 기록)과 반자동(스크립트 보조)의 법적 차이가 있는지?
3. 스냅샷 이미지를 내부 증빙으로 보관하는 행위 자체의 리스크는?

### 2.2 네이버 공식 API 결과의 저장·표시 범위

- 지역 검색 API 응답(업체명, 주소, 카테고리)을 DB에 저장하고,
  등급화된 지표("상위권/중위권/미노출")로 가공해 고객에게 표시합니다.
- **질의:** API 약관상 저장 기간·표시 방식·출처 표기 요건에 제약이 있는지?

### 2.3 의료광고법 관점

- 리포트에는 경쟁 병원의 이름·거리·공개 집계값(리뷰 수 등)이 표 형태로 나옵니다.
- 개선 액션은 운영·정보 정비로 한정하고, 효과 보장·비교우위 단정·환자 후기
  활용 제안을 배제했습니다.
- **질의:**
  1. 경쟁 병원명을 나열·비교하는 리포트가 의료법상 비교광고/비방으로
     해석될 소지가 있는지? (리포트는 광고물이 아닌 원장용 내부 분석자료입니다)
  2. 저희가 원장에게 제안하는 "액션 문구"가 의료광고 심의 대상이 되는 경우가 있는지?
  3. 저희 서비스 자체의 광고 문구(랜딩 페이지)에서 피해야 할 표현 검토

### 2.4 공공데이터 이용

- HIRA 병원정보서비스, 소상공인 상가정보, SGIS를 이용허락범위 내 사용,
  출처·갱신일을 리포트에 표기합니다.
- **질의:** 공공데이터를 유료 서비스에 활용할 때 추가 의무(재배포 제한 등)가 있는지?

### 2.5 약관·개인정보처리방침 작성

- 필요 문서: 서비스 이용약관, 개인정보처리방침, (필요 시) 위탁·제3자 제공 고지
- 특이사항: 환자 개인정보는 수집하지 않음. 수집 정보는 병원 담당자 계정
  (이메일, 병원명, 주소)과 결제 정보에 한정.
- **질의:** 표준 SaaS 약관 대비 의료기관 대상 B2B 서비스로서 추가할 조항이 있는지?
