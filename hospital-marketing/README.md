# 메디랭크 로컬 — 병원 마케팅 경쟁력 진단 SaaS

병원 원장이 내 병원 주소·진료과를 입력하면 반경 500m~2km 안의 경쟁 병원, 인구/상권 수요, 네이버 검색 노출 상태를 종합해 **월간 마케팅 경쟁력(0~100점)** 을 진단하고 개선 액션을 제안하는 SaaS의 설계·프로토타입입니다.

> "우리 병원이 주변 1km 안에서 경쟁 병원 대비 얼마나 잘 검색되고 있는지,
> 공공데이터와 공식 API 기반으로 매달 진단해드립니다."

## 프로토타입 미리보기

GitHub Pages 배포 시: `https://recon9973-lang.github.io/Design-resources-repository/hospital-marketing/`

화면 5개 (기획서 5장):

1. **대시보드 홈** — 경쟁력 점수(노출 40% · 경쟁밀도 25% · 수요입지 20% · 플레이스품질 15%), 반경 선택, 6개월 추이, 이번 달 액션 3가지
2. **지도 분석** — 반경 링 + 경쟁 병원 분포 (같은 진료과/유사 진료과 구분)
3. **키워드 노출** — 공식 API 지표와 월간 참고 스냅샷 분리 표기, 상위권/중위권/미노출 등급
4. **경쟁 벤치마크** — 리뷰 수·평점·사진 수 등 공개 집계값 비교
5. **액션 리포트** — 의료광고 리스크를 제거한 운영·정보 정비 중심 제안 + PDF 저장(인쇄)

## 설계 원칙 (B-safe)

- 공식 API·공공데이터 중심. 실시간 순위 추적, 대량 크롤링, 캡차/차단 우회 없음
- 실제 검색화면 스냅샷은 법무 검토 후 월 1회 참고용으로만, 원본 증빙은 관리자단 전용
- 고객 화면에는 등급화된 안전 요약값만 노출 (`internal_evidence_items.visibility_scope`, DB 트리거로 강제)
- 모든 화면·PDF에 데이터 출처, 기준일, 측정 조건, 디스클레이머 표기
- 추천 액션은 의료법상 안전한 운영 개선 중심 (효과 보장·후기 활용·비방 비교 제외)

## ⚠️ 원장님(운영자)이 직접 해야 할 일

코드/설계로 대신할 수 없는 항목입니다. 실데이터 연동 단계 전까지 준비해 주세요.

1. **API 키 발급**
   - [공공데이터포털](https://www.data.go.kr) 회원가입 → "건강보험심사평가원 병원정보서비스" 활용신청 (무료, 자동승인)
   - [네이버 개발자센터](https://developers.naver.com) 애플리케이션 등록 → 검색(지역) API Client ID/Secret 발급
   - [SGIS 통계지리정보](https://sgis.kostat.go.kr/developer/) 인증키 발급
   - [소상공인시장진흥공단 상가(상권)정보](https://www.data.go.kr/data/15083033/fileData.do) CSV 다운로드 (분기별)
2. **법무 검토 의뢰** — 월간 검색화면 스냅샷 수집 방식(11.2장 체크리스트)과 서비스 약관·개인정보처리방침. 스냅샷 기능은 검토 완료 전까지 비활성 상태로 둡니다.
3. **제품명 확정 및 상표 검색** — 후보: 메디랭크 로컬(프로토타입 가칭) / 병원 상권 랭크 / 메디플레이스 인사이트 / 닥터랭크 맵 / 우리병원 노출진단. [KIPRIS](http://www.kipris.or.kr)에서 상표 선등록 여부 확인
4. **파일럿 원장 5명 섭외** — 1단계 성공 기준(3명 이상 유료 의향)의 검증 대상
5. **사업자/결제 준비** — 요금제(Starter 9~19만원, Pro 29~49만원, Premium 79만원~) 결제 수단(토스페이먼츠 등) 가맹 신청
